# CS50's Introduction to Databases with SQL - Portfolio 🗄️

Welcome to my repository! This workspace contains my solutions, projects, and explorations for **CS50's Introduction to Databases with SQL** (CS50 SQL), a specialized course provided by Harvard University.

## 📖 About This Repository
The purpose of this repository is to showcase my progression and understanding of database management. Throughout the modules, I worked with real-world datasets to model relationships, normalize data, write complex queries, optimize database performance, and scale applications using SQLite, PostgreSQL, and MySQL.

## 🗂️ Course Structure & Folder Mapping

My solutions are organized into folders corresponding to the course's weekly problem sets. Here is the breakdown:

### `Week 0: Querying`
* 📁 **`cyberchase`**: Writing SQL queries to explore a database of episodes from the PBS show Cyberchase.
* 📁 **`views`**: Querying a database about Japanese woodblock prints (36 Views of Mount Fuji).
* 📁 **`players`**: Exploring statistics of Major League Baseball players.
* 📁 **`normals`**: Investigating normal ocean surface temperatures.

### `Week 1: Relating`
* 📁 **`packages`**: Solving a mystery of missing packages using `JOIN`s.
* 📁 **`dese`**: Analyzing open data from the Massachusetts Department of Elementary and Secondary Education.
* 📁 **`moneyball`**: Writing complex queries to find undervalued baseball players using the "Moneyball" philosophy.

### `Week 2: Designing`
* 📁 **`atl`**: Designing a database schema for concourse operations at the Atlanta airport.
* 📁 **`connect`**: Creating a database for a LinkedIn-like professional networking platform.
* 📁 **`donuts`**: Modeling an e-commerce database for Union Square Donuts.

### `Week 3: Writing`
* 📁 **`meteorites`**: Importing historical meteorite landing data from a CSV, cleaning it, and inserting it into a normalized SQL database.

### `Week 4: Viewing`
* 📁 **`census`**: Creating views to aggregate data from the US Census.
* 📁 **`private`**: Decoding a secret message using SQL views and string manipulation.
* 📁 **`bnb`**: Building views for a bed-and-breakfast application to simplify data access.

### `Week 5: Optimizing`
* 📁 **`snap`**: Optimizing database queries for a Snapchat-like app using indexes.
* 📁 **`harvard`**: Analyzing execution plans (`EXPLAIN QUERY PLAN`) and creating indexes to speed up queries on a university database.

### `Week 6: Scaling`
* 📁 **`dont-panic`**: Securing a web application backend by integrating SQL with Python.
